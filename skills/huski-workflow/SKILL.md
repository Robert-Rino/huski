# Huski Workflow

## Purpose
Use this skill to keep the huski-pi package organized and reusable.

## Conventions
- Put extensions in `extensions/`
- Put skills in `skills/`
- Keep each skill self-contained in its own folder

## Extend the package
- Add new extension folders under `extensions/`
- Add new skill folders under `skills/`
- Update `package.json` only if new top-level resource types are added
